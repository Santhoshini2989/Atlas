# Atlas User Personas

## Purpose

This document defines the primary users of Atlas.

Personas help us understand who we are building for and guide product decisions.

---

# Persona 1 — Technical Founder

## Name

Alex

## Role

Technical Founder / Startup Builder

## Company

Early-stage SaaS startup

## Team Size

3-10 people

---

# Background

Alex is building a software product with a small engineering team.

Alex is responsible for:

- Product decisions.
- Technical direction.
- Team coordination.
- Understanding customer needs.

---

# Daily Workflow

A typical day:

Morning:

- Reviews project progress.
- Checks completed tasks.
- Plans priorities.

During the day:

- Talks with customers.
- Discusses features.
- Works with developers.
- Makes technical decisions.

End of day:

- Updates tasks.
- Reviews blockers.
- Plans next steps.

---

# Goals

Alex wants:

- Faster product development.
- Better visibility into progress.
- Clear understanding of what the team is building.
- Less time managing project information.

---

# Problems

Alex struggles with:

## 1. Turning Ideas Into Tasks

Example:

Idea:

"Add team collaboration."

Questions:

- What features are needed?
- How much work is involved?
- Who should do it?
- What should happen first?

---

## 2. Losing Context

Important information exists across:

- Messages.
- Documents.
- Meetings.
- Task systems.

Finding previous decisions takes time.

---

## 3. Planning Takes Too Long

Alex spends hours:

- Writing tickets.
- Creating plans.
- Explaining requirements.

---

# How Atlas Helps Alex

Atlas helps by:

- Converting ideas into structured tasks.
- Summarizing project status.
- Keeping important context organized.
- Assisting with planning.

---

# Persona 2 — Software Engineer

## Name

Sam

## Role

Software Engineer

---

# Background

Sam builds features and solves technical problems.

Sam needs clear requirements to write quality code.

---

# Goals

Sam wants:

- Clear tasks.
- Good technical context.
- Fewer unclear requirements.
- Less time searching for information.

---

# Problems

Sam struggles with:

## Missing Information

Tasks often lack:

- Technical details.
- Expected behavior.
- Acceptance criteria.

---

## Context Switching

Sam jumps between:

- Task trackers.
- Documentation.
- Messages.
- Code.

---

# How Atlas Helps Sam

Atlas provides:

- Better task descriptions.
- AI-generated context.
- Project history.
- Clear requirements.

---

# Persona 3 — Product Manager

## Name

Taylor

## Role

Product Manager

---

# Goals

Taylor wants:

- Better planning.
- Clear priorities.
- Visibility into progress.

---

# Problems

Taylor struggles with:

- Translating ideas into engineering work.
- Tracking progress.
- Communicating changes.

---

# How Atlas Helps Taylor

- Organize product ideas.
- Create structured tasks.

- Track execution.

---

# Persona Summary

| User              | Main Need                              |
| ----------------- | -------------------------------------- |
| Technical Founder | Turn ideas into execution plans        |
| Software Engineer | Understand work clearly                |
| Product Manager   | Organize and track product development |
